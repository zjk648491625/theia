Interface CommentController

A comment controller is able to provide comments support to the editor and provide users various ways to interact with comments.

Optional commenting range provider. Provide a list ranges which support commenting to any given resource uri.

If not provided and emptyCommentThreadFactory exits, users can leave comments in any document opened in the editor.

Optional template

template: CommentThreadTemplate
  • Defined in packages/plugin/src/theia.d.ts:9423

Optional comment thread template information.

The comment controller will use this information to create the comment widget when users attempt to create new comment thread from the gutter or command palette.

When users run CommentThreadTemplate.acceptInputCommand or CommentThreadTemplate.additionalCommands, extensions should create the approriate CommentThread.

If not provided, users won't be able to create new comment threads in the editor.
